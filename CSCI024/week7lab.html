Task 3:

I reviewed the explanatory material. The help menu can be accessed with the question mark "?" command. The "search" command searches the available modules and exploits. The "use" command selects a module for use. The "info" command provides information about a module. The "connect" command will connect to a host/target. The "banner" command will display an ASCII graphic. The "set" command assigns a value to a variable. The "setg" command assigns a value to a global variable. A context-specific variable can be retrieved with "get" and a variable can be cleared/erased with the "unset" command. The "spool" command will save console output to a file. The "save" command records the current values to a file which can be restored later.

Task 4:

I reviewed the explanatory material. The "exploit" module holds individual exploit code. The "payload" module holds code to be when access has been gained. The "auxiliary" module holds code to scan/verify potential targets. The "post" module holds code which loots discovered data or facilitates pivots to another target. An "encoder" module modifies payloads to avoid signature-based detection. The "nop" module is used with buffer overflow or ROP attacks. The "load" command loads different or additional modules.
